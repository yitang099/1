## 触发侦察（按优先级）

### 方式 1: API（推荐，异步）
```bash
curl -s -X POST http://127.0.0.1:18789/scan \
  -H "Content-Type: application/json" \
  -d '{"target":"TARGET_DOMAIN","mode":"full","async":true}'
```
- `mode`: `full`（完整）| `quick`（快速）| `passive`（仅被动子域）
- 查状态: `curl -s http://127.0.0.1:18789/status`
- 健康检查: `curl -s http://127.0.0.1:18789/health`

### 方式 2: 命令行（同步，等结果）
```bash
recon-pipeline.sh TARGET_DOMAIN full
recon-pipeline.sh TARGET_DOMAIN quick
recon-pipeline.sh TARGET_DOMAIN passive
```

### 方式 3: 单工具（针对性任务）
| 任务 | 命令 |
|------|------|
| 子域 | `subfinder -d DOMAIN -silent` |
| 存活 | `httpx -l subs.txt -silent -status-code -title` |
| 端口 | `naabu -list subs.txt -top-ports 1000 -silent` |
| 漏洞 | `nuclei -l alive.txt -severity critical,high -silent` |
| 目录 | `feroxbuster -u URL -w /data/wordlists/common.txt -q` |
| 全量侦察 | `reconftw -d DOMAIN` |
