# xinhe001.lol Phase 4 深挖报告
更新时间: 2026-07-09

探测方式: WebFetch (源 IP curl 仍被 WAF 封锁) + 彩虹发卡源码对照

## 新发现

### 1. [Med] submit.php 三态 oracle 细化

| 请求 | 响应 |
|------|------|
| `?orderid=1` (无 type) | `No type!` |
| `?orderid=1&type=alipay` | `该订单号不存在` |
| `?orderid=999999999&type=alipay` | `该订单号不存在` |

结论: 必须带 `type` 参数才能进入订单存在性判断；与 getshop.php 不同，submit.php **可区分无效订单**（但无法区分存在/不存在——都返回「不存在」除非订单真实存在后跳转支付）。

### 2. [Med] 隐藏商品 SKU — 搜索泄露

`?mod=so&kw=美卡假绑` 返回 **2 条** 首页未展示商品:
- 星星【无脸0违规混合】美卡假绑 — 130元 缺货
- 月亮【无脸0违规混合】美卡假绑 — 140元 缺货

首页只展示「美卡真棒」系列，「美卡假绑」为搜索可见的隐藏 SKU（可能下架或仅链接分销）。

### 3. [High] 支付回调面扩大 — 多通道 notify

| 端点 | Live 响应 |
|------|-----------|
| `other/notify.php` | `No Act` |
| `other/notify.php?act=epay` | HTTP 500 |
| `other/notify.php?act=usdt` | HTTP 500 |
| `other/notify.php?act=alipay` | `No Act` |
| `other/epay_notify.php` | HTTP 500 (GET) |
| `other/alipay_notify.php` | 空 200 |
| `other/wxpay_notify.php` | 空 200 |

比 Phase 2 多确认 alipay/wxpay 独立回调文件存在。USDT 可能走 `notify.php?act=usdt` 或 epay 统一通道。

**攻击链**: 泄露/爆破易支付 `key` → 伪造 `TRADE_SUCCESS` → 任意 `out_trade_no` 标记已付 → `getshop.php` 取卡。

### 4. [Critical-待验证] api.php 彩虹 fork 对照 — 潜在未授权 IDOR

对照 Caihongyunshang/caihong `api.php`，官方彩虹存在 **无鉴权** act:

| act | 危害 |
|-----|------|
| `search` | 按订单 id 返回状态+对接信息 |
| `classlist` | 全部分类 |
| `goodslist` / `goodsdetails` | 商品+库存 |
| `getleftcount` | 库存枚举 |
| `siteinfo` | 站点配置/版本 |

**Live xinhe**: 上述 act 全部 HTTP 500（路由存在，缺参或 PHP 异常）。若配置修复或 POST 带参可能直接泄露订单/卡密。

已测:
- `api.php?act=search&id=1` → 500
- `api.php?act=classlist|siteinfo|goodslist|getleftcount&tid=9` → 500
- `api.php?act=tools&key=test` → 500

建议换 IP POST fuzz: `act=search` + `id=1..N`，`act=goodslist` 空 body。

### 5. [Info] skey 机制 fork 差异（影响爆破策略）

| 版本 | skey 生成 |
|------|-----------|
| 官方彩虹 | `md5(order_id . SYS_KEY . order_id)`，且 `act=query` 直接返回 skey |
| xinhe001 | **用户自设「取卡密码」**（inputvalue2），无服务端公式 |

爆破应优先: 弱口令字典 + 空密码 + `md5(id+hashsalt)` 变体（fork 残留），而非彩虹 SYS_KEY 公式。

### 6. [Med] install.lock 不可读

`GET install/install.lock` → HTTP 500（非早期报告的 200 明文）。重装仍需其他文件删除向量。

### 7. [Low] 信息泄露

- 搜索页客户服务显示 `QQ:123456`（占位或真实）
- 商品页 `?mod=buy&tid=9` 显示 `商品编号：NO.9` — tid 可预测枚举
- `mysid` cookie (32hex) 用于 `inputname=hide` 商品自动填单

### 8. [Med] 语言墙模块清单（需 zh-CN Accept-Language）

WebFetch 无中文头被拦: `mod=order|cutshop|groupshop|seckill|coupon|article`
login.js 确认这些模块存在且 quickreg 可跳转。

### 9. cron.php 密钥排除

`cron.php?key=8d6673bb4bde73830ed11c898186a872` (hashsalt) → 仍「监控密钥不正确」
hashsalt ≠ cronkey。
