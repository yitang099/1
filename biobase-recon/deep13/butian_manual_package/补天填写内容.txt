www.biobasebaby.com 未授权文件上传导致远程代码执行

【厂商名称】

山东博科保育科技股份有限公司

【漏洞URL】
----------------------------------------------------------------------

https://www.biobasebaby.com/index.php/attachment/upload/upload/dir/images/module/formguide.html


----------------------------------------------------------------------
【是否需要登录】
----------------------------------------------------------------------

否


----------------------------------------------------------------------
【漏洞类型】
----------------------------------------------------------------------

WEB漏洞 → 文件上传（或代码执行）


----------------------------------------------------------------------
【危害等级】
----------------------------------------------------------------------

高危


----------------------------------------------------------------------
【漏洞描述】
----------------------------------------------------------------------

山东博科保育科技股份有限公司官网 www.biobasebaby.com 使用的是 YznCMS 系统。我在看前台留言页面的时候发现，页面源代码里直接暴露了文件上传接口地址，路径是 /index.php/attachment/upload/upload/dir/images/module/formguide.html。

这个上传接口没有登录校验，任何人都可以往上面传文件。我试了传 .php 后缀的文件，服务器直接接受了，返回 json 里给了 webshell 的访问路径。然后用浏览器打开这个路径，页面执行了 phpinfo()，显示 PHP Version 7.4.33，说明服务器把上传的 php 文件当代码执行了。

网站页脚可以查到厂商是山东博科保育科技股份有限公司，备案号鲁ICP备15042322号-2。后台登录页显示 BIOBASE管理系统 v1.0.0。

测试过程中我只用了 phpinfo() 做验证，没有动网站数据。


----------------------------------------------------------------------
【复现步骤】
----------------------------------------------------------------------

环境：Windows + Chrome + Burp Suite，不需要账号。

1）打开 https://www.biobasebaby.com/ ，页脚能看到山东博科保育科技股份有限公司和备案号。（见截图1）

2）打开 https://www.biobasebaby.com/index.php/formguide/index/index.html?id=5 ，查看网页源代码，搜索 image_upload_url ，可以看到上传接口地址写在 js 变量里。（见截图2）

3）本地建一个 shell.php，内容：<?php phpinfo(); ?>

4）用 Burp 发 POST 请求到 /index.php/attachment/upload/upload/dir/images/module/formguide.html ，表单字段名 file，上传 shell.php。请求不带 Cookie。服务器返回 json，code 为 0，success 为 1，url 字段是上传后的 php 路径。（见截图3）

5）浏览器访问返回的 url，比如 https://www.biobasebaby.com/uploads/images/20260711/xxxxx.php ，页面输出 phpinfo，版本 7.4.33。（见截图4）

6）补充：FOFA 搜索 body="image_upload_url" && body="formguide" 可以查到同类部署的站点数量。（见截图6，如审核需要）


----------------------------------------------------------------------
【漏洞危害】
----------------------------------------------------------------------

1. 未授权上传 php 文件并执行，属于远程代码执行。
2. 可以读服务器上的配置文件和数据库连接信息。
3. 可以拿到网站后台和前台用户的数据。
4. 可以改网站页面或者挂后门。
5. 这个 ip 上还绑了别的 biobase 相关域名，有风险扩散。


----------------------------------------------------------------------
【修复建议】
----------------------------------------------------------------------

1. 上传接口加登录验证，未登录不允许传。
2. 后缀白名单，只允许 jpg png gif，禁止 php phtml 这些。
3. uploads 目录禁止 php 执行，nginx 里单独配一下。
4. 前台不要把上传地址写在 js 里暴露出来。
5. 升级 YznCMS 到最新版，检查还有没有别的上传点。
6. 排查 uploads 目录有没有已经被传的 webshell，有的话删掉。
